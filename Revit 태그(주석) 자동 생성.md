# 🏷️ Revit Family 기반 자동 태그(주석) 생성 및 최적화 시스템

## 1. 프로젝트 개요 및 비즈니스 임팩트
- **기간:** 2024.04 ~ 2024.08
- **인원:** 1명 (100% 전담)
- **목적 및 성과:** 실무자가 수작업으로 배치해야 했던 도면 내 태그(주석) 작업을 Revit API를 활용해 100% 자동화하여, **도면/작업당 소요 시간을 약 5분 단축**하고 일관된 도면 퀄리티를 확보함.

---

## 2. 기술 스택
- **Language & API:** `C#`, `Revit API`, `AutoCAD API`
- **UI/UX & Tool:** `DevExpress`, `TortoiseSVN`

---

## 3. 핵심 기능 
- **다분야(건축/구조/설비/전기) 통합 태그 자동화:** 벽, 문, 파이프, 트레이 등 객체 특성에 맞는 태그 내용 및 형태 자동 지정.
- **파라미터 기반 자동 연산:** FL(건축 바닥 간 거리), SL(구조 바닥 간 거리) 등 실무 필수 데이터를 자동으로 연산하여 태그에 반영.
- **객체 지향적 방향성 정렬:** Element의 방향성(Vector) 데이터를 추출하여 태그의 기울기 및 배치 각도 자동 동기화.

---

## 4. 💡 핵심 기술 및 트러블슈팅 (Troubleshooting)

### [Issue 1] 비효율적인 메모리 관리로 인한 Transaction 과부하 및 병목 현상 해결
- **문제 (Problem):** 초기 모델링 로직에서 Family 객체 생성을 위해 지역 변수를 파편화하여 사용하고, 중복된 Family 비교 연산이 반복되면서 시스템 작업 속도 저하 및 Transaction 과부하(Overload) 현상이 발생함.
- **해결 (Solution):** 산발적으로 선언되던 지역 변수들을 Class 단위로 구조화(캡슐화)하고, 공통 데이터를 전역 변수로 관리하여 불필요한 중복 연산과 트랜잭션 호출을 최소화하는 객체 지향적 리팩토링을 단행함.
- **결과 (Result):** 코드의 가독성과 유지보수성이 향상되었으며, **300개 태그 생성 기준 기존 10분가량 소요되던 작업 시간을 약 5~6분으로 단축(40 ~ 50% 성능 향상)**시켜 렌더링 딜레이를 완벽하게 해소함.

### [Issue 2] 객체 밀집 구역의 태그 텍스트 중첩으로 인한 가독성 저하 문제 해결
- **문제 (Problem):** 배관 교차점 등 Family 객체가 좁은 공간에 밀집된 구역에서 태그를 자동 생성할 경우, 텍스트끼리 겹치게 되어 도면의 가독성을 심각하게 훼손하고 실무자가 수작업으로 태그를 다시 옮겨야 하는 문제 발생.
- **해결 (Solution):** 생성되는 각 태그의 Bounding Box 좌표(X, Y)를 비교하여 교차(충돌) 여부를 감지하는 로직을 구현함. 겹침이 감지될 경우, 교차된 면적(거리)만큼 객체의 방향성(Vector)을 고려하여 태그를 동적으로 밀어내도록(Displacement) 위치 재조정 알고리즘을 적용함.
- **결과 (Result):** 수십 개의 태그가 생성되는 복잡한 도면에서도 텍스트 중첩률을 0%로 방어하여 도면의 가독성을 완벽히 확보했으며, 배치 후 2차 수작업 수정 과정을 원천적으로 제거하여 진정한 의미의 100% 자동화를 달성함.

> 해당 프로젝트는 Revit API 활용 및 복잡한 태그 자동 배치 규칙 구현 등,  
> 실무 BIM 프로세스에서의 효율적인 주석/정보 관리의 자동화에 중점을 두었습니다.
